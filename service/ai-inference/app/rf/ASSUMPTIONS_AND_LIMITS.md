# Baseline RF — 가정과 한계 (발표용 1장)

## 서비스 구조 (한 줄)

- 사용자에게 보이는 것: **2D floorplan** 위 heatmap·AP·**재질 / 창문 / 가구 영향** 설명 (`docs/SERVICE_RF_ARCHITECTURE.md`).
- **Baseline** = 빠른 preview·반복 시뮬 / **Sionna RT** = 내부 precise 검증 엔진 (3D 계산을 없애는 것이 아니라 UI 전면에서 내림).

## 가정 (모델이 전제로 두는 것)

- **2D 단일 층**: 바닥 평면 위 격자에서 수신 전력만 계산한다.
- **경로 손실**: 로그 거리 기반 path loss 식을 쓴다.
- **벽**: AP–수신점 직선이 벽 중심선과 교차하면, 그 벽의 **재질별 고정 감쇠(dB)**를 더한다. 외곽 벽은 설정에 따라 제외할 수 있다.
- **개구부(opening)**: 동일 직선이 opening 선분과 겹치면, **해당 벽 구간의 감쇠는 적용하지 않는다** (문/창 통과로 가정).
- **다중 AP**: 격자점마다 **strongest RSSI**만 쓴다 (간섭·SINR 없음).
- **재질**: JSON의 **문자열 키**를 `rf_materials`에 정의된 프로파일로 매핑한다.

## 한계 (발표에서 반드시 구분할 것)

- **실제 전파와 다름**: 반사·회절·3D 다중 경로를 모델링하지 않으며, **Sionna RT 등 레이 트레이싱과 동일하지 않다.** 정밀 비교가 필요하면 **내부 Sionna 검증**을 쓴다.
- **입력 형태**: 백엔드 scene graph → RF **자동 빌드 파이프라인은 미연동**이며, **합의된 RF JSON**을 넣는 구조다.
- **재질·보정**: DB의 `material_id`, 운영 **`material_rf_profiles`**, 현장 **캘리브레이션**과는 직접 연동하지 않는다.
- **AP 배치**: 후보는 **휴리스틱**이며 **전역 최적해**를 보장하지 않는다.
- **수치 재현**: 부동소수·numpy·matplotlib 버전에 따라 히트맵·소수 자릿수가 미세하게 달라질 수 있다. **입력 JSON 동일 + 코드 동일**이면 설명 가능한 수준으로 재현된다.
